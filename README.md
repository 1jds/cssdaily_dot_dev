# CSS DAILY Exercises

Exercise answers for CSS Daily

See the project at `cssdaily.dev`

## Thoughts on the platform

- The platform was vibe coded. This is evident from the bugs. For example, for some challenges, text overflows badges/buttons. Some challenges have parts of the preview image clipped off and hidden from sight. The names which the AI comes up with for custom properties are often not consistent (and certainly there is no consistent naming convention between challenges day to day). The mechanism for comparing the user output to the target doesn't appear to be particularly usable, since it is (in my experience anyway) almost impossible to reach anywhere near 100% (I have not tried resorting to absolutely positioning everything and setting fixed widths and heights, etc.)
- This raises for me broader questions about vibe-coding, and what place it can play in production code without human developer intervention.
- I have only found one easy challenge so far, and no hard ones. They are all seemingly 'medium'.
- There is repetition in the sorts of targets (e.g. multiple weather widget type targets).
- The range of CSS skills which can be practiced with these challenges is limited. For example, users will get constant practice with using flex-box, grid, border-radius, font-size, text-transform, etc. But will get no practice in many other CSS features. So there are diminishing returns to continuing to complete these challenges after a while.
- It would be nice to be able to jump to the challenge for a specific date.
- There are sometimes items like checkboxes in the challenges which are `<div>` or `<span>` elements in the html. This is fine, but it would be nice if the challenge was a little more realistic, rather than just being a _visual_ reproduction exercise.
- One tends towards pixel-pushing and absolute values to achieve a high-score, but real web development should be responsive and not used fixed values wherever possible...
- Nevertheless, I had fun completing several of these challenges, and may still do more.
